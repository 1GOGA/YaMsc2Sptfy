## 🎯 Что вообще это?

**YaMsc2Sptfy** - это веб-приложение, которое переносит музыкальные плейлисты из Яндекс Музыки в Spotify.

✨ **Что оно делает:**
- Берет ссылку на публичный плейлист Яндекс Музыки
- Ищет каждый трек на Spotify
- Создает новый плейлист в вашем Spotify аккаунте
- Автоматически добавляет найденные треки

**Очень просто:** ссылка → плейлист готов! 🎉

## 🎵 Шаг 2: Создайте Spotify приложение

**Это нужно для получения ключей доступа.**

### 2а. Откройте Spotify Developer Dashboard

Перейдите на: https://developer.spotify.com/dashboard

**НЕ забудьте войти в Spotify аккаунт!** (или создайте новый)

### 2б. Примите условия

Послеログина вы увидите форму - примите условия и нажмите "Accept"

### 2в. Создайте приложение

1. Нажмите или найдите кнопку **"Create an App"**
2. Дайте название: `YaMsc2Sptfy` (или любое)
3. Примите условия
4. Нажмите **"Create"**

### 2г. Получите учетные данные

На странице приложения вы увидите:
- **Client ID** - скопируйте это
- **Client Secret** - скопируйте это

⚠️ **Храните эти значения в безопасности! Не читайте посторонним!**

### 2д. Добавьте Redirect URI

1. На той же странице нажмите **"Edit Settings"**
2. Найдите секцию **"Redirect URIs"**
3. В поле введите:
   ```
   http://127.0.0.1:8000/auth/callback
   ```
4. Нажмите **"Add"**
5. Нажмите **"Save"**

✅ **Готово!** Spotify приложение создано.

## 📝 Шаг 3: Настройте .env файл

В папке проекта найдите файл `.env.example` - это шаблон.

### 3а. Скопируйте файл

Откройте командную строку в папке проекта и выполните:

```bash
copy .env.example .env
```

**Или вручную:**
1. Откройте `.env.example` в текстовом редакторе (Блокнот, VS Code и т.д.)
2. Сохраните как `.env` в той же папке

### 3б. Отредактируйте .env

Откройте файл `.env` и заполните:

```env
SPOTIFY_CLIENT_ID=ВСТАВЬТЕ_ВАШ_CLIENT_ID
SPOTIFY_CLIENT_SECRET=ВСТАВЬТЕ_ВАШ_CLIENT_SECRET
REDIRECT_URI=http://127.0.0.1:8000/auth/callback
DEBUG=True
```

**Пример:**
```env
SPOTIFY_CLIENT_ID=abcd1234567890efghij
SPOTIFY_CLIENT_SECRET=xyz9876543210abcdef
REDIRECT_URI=http://127.0.0.1:8000/auth/callback
DEBUG=True
```

⚠️ **Важно!**
- Используйте `127.0.0.1` (НЕ localhost!)
- Не добавляйте кавычки вокруг значений
- Сохраните файл

## 🌍 Развертывание онлайн

Если хотите поделиться ссылкой с друзьями, нужно развернуть онлайн.

**Проблема:** Spotify требует HTTPS в production, а 127.0.0.1 работает только локально.

**Решение:** Развернуть на Railway.app (бесплатно, HTTPS автоматически)

### Быстрое развертывание на Railway

1. **Перейдите на https://railway.app**
2. **Нажмите "Sign Up"**
3. **Выберите "Login with GitHub"** (или создайте GitHub аккаунт)
4. **После логина откройте свой проект на GitHub:**
   - https://github.com/1GOGA/YaMsc2Sptfy
5. **На Railway нажмите "New Project"**
6. **Выберите "Deploy from GitHub repo"**
7. **Найдите `YaMsc2Sptfy` и нажмите**
8. **Railway начнет развертывание...**

### Когда развернется

1. **Откройте "Settings" → "Domain"**
2. **Скопируйте ваш URL**, например:
   ```
   https://yamsc2sptfy-production.up.railway.app
   ```

3. **Добавьте переменные окружения:**
   - "Variables" → добавьте ВСЕ из `.env`
   - Но для REDIRECT_URI используйте новый URL:
     ```
     REDIRECT_URI=https://yamsc2sptfy-production.up.railway.app/auth/callback
     ```

4. **Обновите Spotify Dashboard:**
   - https://developer.spotify.com/dashboard
   - Edit Settings → Redirect URIs
   - Замените на: `https://yamsc2sptfy-production.up.railway.app/auth/callback`

5. **Откройте свой сайт:**
   ```
   https://yamsc2sptfy-production.up.railway.app
   ```

✅ **Готово! Сайт живой!**

Теперь можете поделиться ссылкой с друзьями.

## 🔧 Решение проблем

### ❌ "ModuleNotFoundError" или "No module named..."

**Решение:** Не установлены зависимости
```bash
pip install -r requirements.txt
```

### ❌ "Port 8000 already in use"

**Решение:** Порт занят другой программой

Вариант 1:
```bash
# Измените порт в .env
# Замените на:
# APP_PORT=8001
# Потом откройте http://127.0.0.1:8001
```

Вариант 2:
```powershell
# Найдите процесс на порту 8000
netstat -ano | findstr :8000

# Завершите процесс (замените ID на реальный)
taskkill /PID ID /F
```

### ❌ "Redirect URI mismatch"

**Решение:** URL в Spotify Dashboard не совпадает с .env

1. Откройте https://developer.spotify.com/dashboard
2. Edit Settings
3. Проверьте что Redirect URI точно совпадает с REDIRECT_URI в .env
4. Обновите страницу приложения

### ❌ "Cannot connect to Spotify"

**Решение:** Проверьте SPOTIFY_CLIENT_ID и SPOTIFY_CLIENT_SECRET
- Откройте .env
- Удостоверьтесь что они скопированы правильно (без пробелов)
- Сохраните и перезагрузите сервер (Ctrl+C и потом python main.py снова)

### ❌ "Мало треков найдено на Spotify"

**Это нормально!** Трудно найти ВСЕ треки потому:
- Spotify не имеет всех треков (особенно редких)
- Имена артистов отличаются (например "feat." может быть по-другому)
- Инди-исполнители могут быть недоступны

**Совет:** Попробуйте популярные плейлисты сначала.

### ❌ "Сайт не открывается"

Проверьте:
1. Сервер запущен? (видите "running on http://...")
2. Правильный адрес? (http://127.0.0.1:8000 - без localhost!)
3. Нет ошибок в командной строке?

### ❌ "Плейлист Яндекса не найден"

Проверьте:
1. Плейлист публичный (не приватный)
2. Ссылка скопирована правильно
3. Плейлист еще существует
